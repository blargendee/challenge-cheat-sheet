CSS Position Reference

Position Values

Value

Description

Examples

static

Default. Static positioned elements are not affected by the top, bottom, left, and right properties.

An element with position: static; is not positioned in any special way; it is always positioned according to the normal flow of the page.

 Show me

relative

An element with position: relative; is positioned relative to its normal position.

Setting the top, right, bottom, and left properties of a relatively-positioned element will cause it to be adjusted away from its normal position. Other content will not be adjusted to fit into any gap left by the element.

 Show me

fixed

An element with position: fixed; is positioned relative to the viewport, which means it always stays in the same place even if the page is scrolled. The top, right, bottom, and left properties are used to position the element.

A fixed element does not leave a gap in the page where it would normally have been located.

 Show me

absolute

An element with position: absolute; is positioned relative to the nearest positioned ancestor (instead of positioned relative to the viewport, like fixed).

However, if an absolute positioned element has no positioned ancestors, it uses the document body, and moves along with page scrolling.

Note: Absolute positioned elements are removed from the normal flow, and can overlap elements.

 Show me

sticky

An element with position: sticky; is positioned based on the user's scroll position.

A sticky element toggles between relative and fixed, depending on the scroll position. It is positioned relative until a given offset position is met in the viewport - then it "sticks" in place (like position: fixed).

Examples

static

The green box is part of the flow of the document and takes up space such that other objects will place themselves around it.

relative

The green box is part of the flow of the page and takes up space, but is moved relative to the space it would have occupied. It does not take up space in the new position, but still takes up space where it would have been placed.

fixed

The green box in the lower right corner of the screen is fixed into that position and will not move no matter how nicely you ask it to.

absolute

The green box is removed from the flow of the page and placed exactly where it is told to be (in this case, it inherits its top position (relative to the viewport) from the parent container and is a small distance away from the left side of the screen). Since it's been removed from the page flow, it doesn't technically belong in the scroll window despite being a child of that element. It also does not take up space and will overlap elements if they happen to be in the way.

Try shrinking your browser window to see this effect.
